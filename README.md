# 💎⛏️ Minecraft API

Minecraft API is a unified Next.js project that serves both:

- A read-only REST API for Minecraft entities
- A frontend explorer (information pages, terminal, long-form docs and API docs)

> This project is **unofficial** and not affiliated with Mojang.

## Stack

- Next.js 15
- React 19
- TypeScript
- TailwindCSS

## Quick Start

```bash
pnpm install
pnpm dev
```

Open http://localhost:3000.

## Project Structure

- `app/` Next.js App Router entries for the UI and API
- `app/api/` read-only API routes and generated OpenAPI document
- `app/docs/` dedicated interactive API documentation route
- `app/documentation/` long-form project documentation route
- `app/information/` entity-specific information routes
- `src/api/` handler orchestration, OpenAPI generation and low-level response helpers
- `src/components/` shared UI building blocks and the API explorer
- `src/components/information/` reusable entity list/card components
- `src/contexts/` client context providers
- `src/hooks/` client hooks for state, filters, tooltips and background config
- `src/layout/` shared layout wrapper
- `src/services/` browser-side data fetching hooks
- `src/types/` domain models and shared type declarations
- `src/utils/` constants and small presentation helpers
- `src/views/` page-level client views
- `data/` static JSON datasets used by the API
- `public/information/` entity images and information assets

## API Endpoints

Base: `/api`

### Entities

- `/api/advancements`
- `/api/biomes`
- `/api/blocks`
- `/api/items`
- `/api/mobs`
- `/api/structures`

### Generic Subroutes

For each entity path above:

- `/` list all
- `/count` total count
- `/keys` entity keys
- `/:id` entity by id
- `/:id/image` image binary
- `/all/:key` all entries with key
- `/all/:key/:value` filter by key/value
- `/:id/:key` key value by id

### Documentation Endpoints

- `/docs` dedicated interactive API explorer
- `/documentation` long-form project documentation page
- `/api/openapi` generated OpenAPI 3.1 document used by the in-app docs

## Frontend Routes

- `/` home

- `/information/advancements`

## Performance Notes

- API data loading uses async file I/O with in-memory cache
- API responses and image responses include cache headers
- Information pages use progressive rendering for large datasets
- Tooltip updates are throttled with `requestAnimationFrame`
